# Appinn 发帖发布

## 目标

将工具介绍整理为可发布到小众软件论坛的帖子，并尽量提高通过率与可读性。

## 固定规则（必须满足）

1. 先按规则自检：<https://meta.appinn.net/t/topic/43728>
2. 标题不要出现“免费”字样，除非全流程实测确实免费；如测试中出现付费，视为不合规。
3. 禁止标题党：标题和正文必须真实、克制、信息一致。
4. 需要注册才能使用时，必须在正文明显位置说明。
5. 必须保留官网链接。
6. 已发布内容应直接编辑，不要删除后重发。
7. 帖子语言尽量简洁，排版整洁，优先“易读、可信、可验证”。
8. 必须严格按照排版范例进行版式组织与信息分段：<https://meta.appinn.net/t/topic/23842>（不允许仅套模板而忽略范例的段落节奏与可读性要求）。

## 输出流程

按以下顺序输出内容：

1. 先给「合规检查结果」：逐条列出通过/待补充。
2. 再给「可直接发布的帖子正文」。
3. 最后给「发布前 30 秒复核清单」。

## 帖子模板（默认结构）

按下面结构生成，不要省略“注册/付费说明”：

```markdown
# {工具名}（{一句定位，避免夸张词}）

{一句话简介：这是什么，解决什么问题}

## 软件名称
{中文名 / 英文名}

## 官网
{官网 URL}

## 平台
- Windows
- macOS
- Linux
- Web
- iOS / Android

## 推荐类型
【开发者自荐】/【编辑推荐候选】/【体验分享】

## 使用门槛（必写）
- 是否需要注册：{需要 / 不需要}
- 是否付费：{免费 / 部分付费 / 订阅制 / 买断}
- 付费触发点：{例如导出高清、批量处理、云同步等；无则写“未发现”}

## 一句话简介
{20~40 字，避免营销话术}

## 应用简介
1. {核心功能 1}
2. {核心功能 2}
3. {核心功能 3}
4. {差异点或亮点}

## 适用人群
- {人群 A}
- {人群 B}

## 已知限制（建议写）
- {限制 1}
- {限制 2}

## 演示内容（可选）
{如需嵌入视频，将视频链接单独放一行；支持 mp4/Bilibili/YouTube}
```

## 写作约束

- 使用陈述句，不用“最强、神器、必备、吊打”等词。
- 每段尽量短句，列表优先，减少大段堆叠文本。
- 未实际验证的信息标记为“待确认”，不要猜测。
- 如果产品需要登录、邀请码、地区限制，必须前置说明。
- 默认使用白话表达，优先普通用户能看懂的词汇；避免专业术语、英文缩写与行话。
- 若必须出现专业词，紧跟一句通俗解释（例如“API Key：调用服务时用的密钥”）。
- 当用户要求“白话版/通俗版/小白可懂”时，需对标题、副标题、一句话简介、应用简介整体改写，不得仅替换个别词语。

## 排版执行规范（强制）

- 以排版范例为基准组织内容顺序、标题层级、段落长度与留白节奏：<https://meta.appinn.net/t/topic/23842>。
- 标题、一级标题、二级标题之间需有清晰分隔；避免连续大段文字，优先短段 + 列表。
- “使用门槛（注册/付费/触发点）”必须前置且醒目，不得埋在文末。
- 链接（官网、视频）单独成行，避免与说明句混排导致可读性下降。
- 输出前必须自检：若与排版范例风格冲突，先重排再输出，不直接提交。

## 快速复核清单（发布前）

- [ ] 标题无“免费”字样或已完整验证免费
- [ ] 正文明确写出是否需要注册
- [ ] 正文明确写出付费模式与触发点
- [ ] 官网链接可访问
- [ ] 无标题党/夸张承诺
- [ ] 视频链接为单独一行（如果有）
- [ ] 排版整洁，一级二级标题清晰
- [ ] 已严格对齐排版范例的结构与阅读节奏（参考：<https://meta.appinn.net/t/topic/23842>）
- [ ] 全文用词通俗，小白可读；如有专业词，已补一句解释
